This is a model accuracy / training problem.

If:

Food waste → predicted as metal

Plastic → predicted as glass

Categories totally mismatched

Then the issue is inside one of these areas:

🎯 MAIN REASONS WHY THIS IS HAPPENING
✅ 1️⃣ Your Model Was Not Properly Trained (Most Common)

If you used:

MobileNetV2

but:

Trained only few epochs

Used small dataset

Dataset imbalance

Wrong labels

No fine-tuning

Then the model doesn't actually understand waste categories.

It is just guessing based on shapes or colors.

Example:
Food waste brown color → model thinks rusted metal
Plastic transparent → model thinks glass

✅ 2️⃣ Your Dataset Labels Are Incorrect

Check your dataset folders carefully:

dataset/
   organic/
   recyclable/
   hazardous/
   non_recyclable/

If images are placed in wrong folders even 10–15% wrong → model becomes confused.

This is very common mistake.

✅ 3️⃣ Class Index Mapping Is Wrong

This is VERY IMPORTANT.

Check your class labels order during training.

Example during training:

class_indices:
{
 'glass': 0,
 'metal': 1,
 'plastic': 2,
 'organic': 3
}

But in inference you are mapping like:

labels = ['organic','plastic','metal','glass']

🔴 This will give completely wrong output.

Even if model predicts correctly internally, your label mapping shows wrong class.

This is the MOST COMMON inference error.

✅ 4️⃣ Overfitting / Underfitting

If:

Training accuracy 95%

Validation accuracy 40%

Then model memorized training images only.

It cannot generalize to new uploaded images.

✅ 5️⃣ Poor Quality Dataset

If dataset:

Too small (< 500 images total)

Low resolution

Same background

Similar lighting

Then model learns background instead of object.

🔥 HOW TO FIX IT PROPERLY
🧠 STEP 1: Check Class Mapping Immediately

In your training code, print:

print(train_generator.class_indices)

Save this dictionary.

Now in inference:

labels = {0: 'glass', 1: 'metal', 2: 'plastic', 3: 'organic'}

Make sure mapping EXACTLY matches training.

If wrong → fix immediately.

🧠 STEP 2: Improve Dataset (Very Important)

Minimum recommended:

1000+ images per class

Different angles

Different lighting

Different backgrounds

Real-world messy waste

If you trained on clean studio images → real garbage images will fail.

🧠 STEP 3: Use Proper Transfer Learning

Correct way with MobileNetV2:

base_model = MobileNetV2(
    weights='imagenet',
    include_top=False,
    input_shape=(224,224,3)
)

base_model.trainable = False

x = GlobalAveragePooling2D()(base_model.output)
x = Dense(128, activation='relu')(x)
output = Dense(4, activation='softmax')(x)

model = Model(base_model.input, output)

Then train 10–20 epochs.

After that:

Unfreeze last 20 layers and fine-tune.

🧠 STEP 4: Check Confusion Matrix

After training:

from sklearn.metrics import confusion_matrix

See which class is confusing with which.

Example:

Organic → confused with metal
Plastic → confused with glass

Then you know where dataset needs improvement.

🧠 STEP 5: Add Data Augmentation (Only in Training)

Use:

rotation

zoom

brightness

horizontal flip

This improves generalization.

🧠 STEP 6: Balance Dataset

If:

Organic images = 2000
Metal images = 300

Model will predict organic more often.

Make dataset balanced.

🧠 STEP 7: Increase Training Quality

Use:

epochs = 25–40
batch_size = 32
optimizer = Adam(learning_rate=0.0001)

Monitor validation accuracy.

🎓 Academic-Level Fix

Instead of only classification, consider:

Upgrade to object detection (YOLO).

Because waste images often contain multiple mixed objects.

Classification struggles when:

Food waste + plastic bag together → model confused.
